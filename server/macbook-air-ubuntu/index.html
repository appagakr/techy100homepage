맥북에어 2011 모델에 Ubuntu 및 Android-x86 멀티부팅 설치

June 1, 2022

맥북에어 2011 모델에 Ubuntu 및 Android-x86 멀티부팅 설치

2011년 경에 구입한 MacBook Air (11-inch, Mid 2011)를 안쓰고 방치한지 오래입니다. 4GB 램에 128GB 디스크에 아직도 큰 문제없이 작동은 합니다. 묵혀두기에는 아까워서 ubuntu 22.04 설치하여 테스트용으로도 쓰고 Android-x86도 멀티부팅 가능하게 설치하여 안드로이드도 사용하고자 설치해 보았습니다.

별도의 파티션에 두 OS를 설치하는 것이 아닌 우분투에 rpm으로 안드로이드를 설치하여 멀티부팅으로 선택하여 사용하도록 설치한합니다.

우분투 설치용 USB 드라이브 만들기

Ubuntu에서 데스크탑용 ISO 이미지 파일 다운받는다. 최신 버전인 Ubuntu 22.04를 다운로드 합니다.

다운로드 받은 이미지 파일을 부팅용 usb 드라이브로 만들 때 rufus를 이용하여 만듭니다. USB 드라이브는 최소 8GB 이상 용량이 필요합니다.

rufus로 드라이브 제작시에 ISO, dd 방식으로 만들 것이냐는 질문을 물어보는데 dd 방식으로 제작하니 정상적으로 부팅되었습니다.

맥북에 우분투 설치하기

usb 부팅드라이브를 꽂은 후 전원을 키고 화면이 켜지면 바로 option 키를 계속 누르고 있으면 부팅디스크 선택 화면이 표시됩니다. usb 드라이브(EFI Boot)를 선택하면 우분투 부팅 화면으로 진입하게 됩니다.

설치 완료 후 아래처럼 패키지를 최신으로 업데이트 합니다.

sudo apt update
sudp apt -y upgrade

안드로이드 설치

Android-x86에서 최신 설치 파일을 다운받아서 설치하도록 합니다.

ISO를 받아서 별도의 OS로 설치하는 것이 아니라 rpm 설치 파일을 받아서 설치하면 됩니다. 리눅스에 안드로이드를 멀티부팅 가능하도록 설치를 합니다.

현재 최신 안정 버전인 lineage OS cm-x86-14.1-r5.x86_64.rpm 파일을 다운로드 합니다. 다운로드 경로는 릴리즈노트를 참고하면 됩니다.

rpm 패키지를 우분투용 패키지로 바꿔주는 alien 툴을 먼저 설치 후 alien으로 다운로드 받은 파일을 설치하도록 합니다.

sudo apt install alien
sudo alien -ci cm-x86-14.1-r5.x86_64.rpm

패키지를 변환하는 작업이 시간이 걸리는지 설치 완료되는데 제 경우에는 10분 이상 소요되었습니다.

설치 완료되면 /cm-x86-14.1-r5 폴더가 생성되고 모든 파일이 하위에 설치됩니다.

qemu-android 명령으로 리눅스에서 바로 실행이 가능합니다. qemu-system 패키지를 먼저 설치하고 실행하도록 합니다.

sudo apt install qemu-system
sudo qemu-android

grub 편집하여 부팅메뉴 표시하기

우분투는 디폴트로 grub 부팅메뉴가 표시되지 않도록 세팅되어 있습니다. /etc/default/grub 설정 파일을 수정하여 표시되도록 바꿔줍니다.

sudo vim /etc/default/grub

아래처럼 값을 변경합니다. 부팅시 menu 형태로 10초 동안 표시하도록 설정한 것입니다.

/etc/default/grub
GRUB_TIMEOUT_STYLE=menu
GRUB_TIMEOUT=10

파일 변경 후에는 아래 명령을 실행해야 변경 사항이 반영됩니다.

sudo update-grub

재부팅하면 grub 메뉴에서 안드로이드를 선택하여 부팅할 수 있습니다.

grub 편집하여 디폴트로 안드로이드로 부팅하기

grub 설정에서 GRUB_DEFAULT=0 부분이 부팅 시 디폴트로 사용하는 커널을 설정하는 부분입니다. grub 부팅화면에서 안드로이드가 세번째로 표시되고 있으니 순서(인덱스)로 2가 됩니다. GRUB_DEFAULT=2로 설정을 수정하고 재부팅하면 자동으로 안드로이드로 부팅됩니다.

$ sudo vim /etc/default/grub

GRUB_DEFAULT=2
GRUB_TIMEOUT_STYLE=menu
GRUB_TIMEOUT=3

$ sudo update-grub
$ reboot

안드로이드 설치 제거

아래 명령으로 설치한 패키지를 제거 가능하며, 설치 폴더가 완전히 삭제되지 않는 경우가 있으니 깨끗히 지워주면 됩니다.

sudo apt-get remove android-x86
sudo rm -rf /cm-x86-14.1-r5
