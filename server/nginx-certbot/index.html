certbot으로 nginx용 SSL 설치

January 21, 2024

certbot으로 nginx용 SSL 설치

certbot을 이용하여 nginx 서버에 SSL 인증서 설치하는 방법입니다. 서버는 ubuntu 22.04 입니다.

certbot을 먼저 설치하고 SSL 인증서를 발급하여 설정합니다.

certbot으로 발급받은 인증서는 3개월간 유효합니다. crontab으로 주기적으로 SSL 인증서를 갱신하는 certbot 명령어를 실행하도록 하여 인증서가 만료되지 않도록 합니다.

certbot 설치

certbot 설치가이드에 따라 먼저 certbot을 설치합니다.

snap 명령어가 필요한데 ubuntu 22.04 버전에는 snapd 패키지가 이미 설치되어 있습니다.

OS에 포함되어 기 설치된 패키지를 먼저 제거합니다.

sudo apt-get remove certbot

snap으로 certbot을 설치하고 /usr/bin/ 하위로 링크 합니다.

sudo snap install --classic certbot
sudo ln -s /snap/bin/certbot /usr/bin/certbot

nginx용 인증서 발급

80 포트로만 설정된 nginx 설정 파일 /etc/nginx/sites-available/default은 미리 용도에 맞게 설정되어 있습니다.

/etc/nginx/sites-available/default
# admin server
upstream admin {
    server 127.0.0.1:1448;
}
server {
    # Listen HTTP
    listen 80;
    server_name yourhost.com;

    # Proxy Config
    location / {
        proxy_pass http://admin;
        proxy_http_version 1.1;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Server $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Host $http_host;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_pass_request_headers on;
    }
}

현재 nginx에 설정된 도메인(yourhost.com)으로 인증서를 최초로 발급합니다. certbot --nginx 명령으로 nginx 용으로 발급을 진행합니다.

certbot 실행 후 nginx 설정 파일이 수정됩니다.

/etc/nginx/sites-available/default
# admin server
upstream admin {
    server 127.0.0.1:1448;
}
server {
    # Listen HTTP
    server_name yourhost.com;

    # Proxy Config
    location / {
        proxy_pass http://admin;
        proxy_http_version 1.1;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Server $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Host $http_host;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_pass_request_headers on;
    }

    listen 443 ssl; # managed by Certbot
    ssl_certificate /etc/letsencrypt/live/yourhost.com/fullchain.pem; # managed by Certbot
    ssl_certificate_key /etc/letsencrypt/live/yourhost.com/privkey.pem; # managed by Certbot
    include /etc/letsencrypt/options-ssl-nginx.conf; # managed by Certbot
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem; # managed by Certbot

}


server {
    if ($host = yourhost.com) {
        return 301 https://$host$request_uri;
    } # managed by Certbot


    listen 80;
    server_name yourhost.com;
    return 404; # managed by Certbot
}

cron을 이용한 certbot 인증서 자동갱신

cron을 이용하여 certbot 인증서가 자동갱신되도록 설정합니다.

갱신 전에 인증서를 체크할 수 있는 명령어 입니다.

갱신 테스트

갱신이 제대로 진행가능한지 확인할 수 있습니다. --dry-run은 테스트만 하는 옵션으로 제외하고 명령을 실행하면 인증서가 갱신 됩니다.

$ sudo certbot renew --dry-run

cron 작업 만들기

/etc/cron.d 폴더에 certbot 파일을 만들어 한달에 한번 실행되는 갱신용 스크립트를 작성합니다.

whereis 명령으로 certbot 명령이 /usr/bin/ 폴더에 위치하는 것을 확인할 수 있습니다.

certbot 파일에 실행시 사용할 쉡과 경로를 설정합니다. 매월 1일, 15일 자정에 root 계정으로 갱신 명령을 실행하도록 설정합니다.

$ whereis certbot
certbot: /usr/bin/certbot /snap/bin/certbot
$ cd /etc/cron.d
$ sudo vi certbot

SHELL=/bin/sh
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin
0 0 1,15 * * root certbot renew --nginx

cron 작업 로그는 /var/log/syslog 파일로 확인할 수 있습니다.

view /var/log/syslog
